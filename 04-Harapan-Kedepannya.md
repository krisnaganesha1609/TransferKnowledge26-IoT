# Harapan Pengembangan Divisi

## Pendahuluan

Dokumen ini berisi harapan dan arah pengembangan Divisi Software for IoT untuk periode-periode berikutnya.

---

# Meningkatkan Kualitas Teknis

Target ke depan:

- (More) Clean Architecture
- Testing
- Semantic Versioning
- Branching Utilizing
- CI/CD
- Documentation

---

# Meningkatkan Kualitas Project

Project yang dikembangkan harus:

- Memiliki tujuan jelas
- Dapat digunakan
- Memiliki dokumentasi
- Mudah dikembangkan

---

# Meningkatkan Budaya Belajar

Divisi harus menjadi tempat belajar.

Bukan hanya tempat mengerjakan tugas.

Budaya yang perlu dibangun:

- Sharing session
- Code review
- Pair programming
- Internal workshop

---

# Meningkatkan Kolaborasi

Divisi software tidak dapat berjalan sendiri.

Kolaborasi perlu diperkuat dengan:

- Divisi hardware
- Divisi riset
- Divisi lainnya

---

# Membangun Regenerasi yang Baik

Kesuksesan divisi tidak diukur dari satu periode.

Kesuksesan divisi diukur dari kemampuannya menghasilkan generasi berikutnya yang lebih baik.

Karena itu:

- Dokumentasi harus dijaga
- Knowledge sharing harus rutin
- Anggota baru harus dibimbing

---

# Penutup

Harapan terbesar bukanlah menghasilkan project terbanyak.

Harapan terbesar adalah membangun divisi yang mampu terus berkembang bahkan ketika seluruh pengurus saat ini sudah tidak lagi menjabat.
