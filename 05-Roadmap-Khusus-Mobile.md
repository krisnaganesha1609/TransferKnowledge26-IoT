# Roadmap dan Standar Divisi

## Tujuan Dokumen

Dokumen ini menjadi panduan bagi anggota baru untuk mengetahui arah belajar dan standar kerja yang digunakan dalam divisi.

---

# Roadmap Pembelajaran

## Tahap 1 — Fundamental

- Dasar Pemrograman
- Git dan GitHub
- Dart Fundamental

Target:
Mampu memahami project sederhana.

---

## Tahap 2 — Flutter

- Widget
- Navigation
- Form
- API

Target:
Mampu membuat aplikasi sederhana.

---

## Tahap 3 — State Management

- Riverpod 3.0
- Provider Lifecycle
- AsyncValue

Target:
Mampu mengelola state aplikasi dengan baik.

---

## Tahap 4 — IoT Integration

- MQTT
- REST API
- WebSocket

Target:
Mampu menghubungkan aplikasi dengan perangkat IoT.

---

## Tahap 5 — Software Architecture

- Clean Architecture
- Repository Pattern
- Dependency Injection

Target:
Mampu membangun project yang scalable.

---

# Standar Git

Branch:

- main
- development
- feature/*
- fix/*

Commit:

feat:
fix:
refactor:
docs:
test:

---

# Standar Dokumentasi

Setiap project minimal memiliki:

- README
- Installation Guide
- Architecture Overview
- API Documentation

---

# Standar Project

Project dianggap selesai apabila:

- Fitur berjalan
- Tidak ada bug kritis
- Dokumentasi tersedia
- Dapat dijalankan oleh anggota lain

---

# Penutup

Tujuan utama roadmap dan standar ini bukan membatasi kreativitas.

Tujuannya adalah memastikan seluruh anggota bergerak ke arah yang sama dan memiliki fondasi yang kuat dalam pengembangan perangkat lunak untuk IoT.
